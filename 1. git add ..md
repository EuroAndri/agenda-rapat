**1. Cek Perubahan**



• Bagian CHANGES akan menampilkan file yang sudah kamu edit, misalnya DatabaseS....



**2. Stage Changes (Tambah ke Staging Area)**



• Klik tanda + di samping nama file (atau klik kanan → Stage Changes).



• File akan pindah dari CHANGES ke STAGED CHANGES.



**3. Tulis Commit Message**



• Di kotak Message (Ctrl+Enter) tulis deskripsi singkat perubahanmu.

Contoh: Perbaikan query Database



**4. Commit**



• Klik tanda centang ✔ Commit atau tekan Ctrl+Enter.



• Ini menyimpan perubahan ke lokal Git.



**5. Push ke Remote Repository (misalnya GitHub/GitLab)**



• Klik ikon ↻ (synchronize changes) di bagian atas atau pilih menu ... → Push.



• Pastikan sudah login dan repository Git remote (misalnya GitHub) sudah terhubung.



Setelah itu perubahanmu akan masuk ke repository di GitHub (atau remote lain).
